### 操作系统发展历程

#### 未配置操作系统的计算机系统

```
	
	第一代计算机 真空管
	
	采取人工操作方式
		通过纸带卡片 将程序和数据输入到计算机 
		用户独占全机 CPU等待人工操作
	
	脱机输入输出方式
		事先把纸带转股纸带输入机中 在外围机控制下 把纸片上的数据输入到磁带上 CPU需要这些程序和数据时 从磁带上将其高速调入内存 减少了CPU的空闲时间 缓和人机矛盾(装卸纸带)
	
```

#### 单道批处理系统

```
	
	第二代计算机 晶体管
	
	批处理系统旨在提高系统资源的利用率和系统吞吐量
	
	单道批处理系统
		//解决人机矛盾和CPU与I/O设备不匹配的矛盾 形成的
		在系统配置监督程序 让作业一个接一个的调入内存 成批量处理 但是只有一道作业
		特征 自动性 顺序性 单道性(内存只有一道程序运行)
		不能重复利用系统资源 读取I/O设备(低俗读取)时CPU处于等待状态
	
```

#### 多道批处理系统

```
	
	第三代计算机 小规模集成电路
	
	多道批处理系统
		//为了进一步提高资源的利用率和吞吐量 解决单道批系统的问题
		作业先存放在外存排列(形成后备队列) 之后选取若干个作业调入内存 共享CPU资源 在运行A时利用I/O时间调度运行B 使多道程序交替运行 可以保持CPU处于忙碌状态
		资源利用率高 系统吞吐量大 平均周转时间长 
		无交互能力 修改和调试程序不方便
		多批道处理系统需要解决的问题(了解)//通过向计算机系统增加一组软件 解决了以下问题
			处理机争用问题
			内存分配和保护问题(不能因为某程序出现异常 破坏其它程序)
			I/O设备分配问题
			文件的组织和管理问题
			作业管理问题
			用户与系统的接口问题
		
	
```

### 操作系统的基本特性

```
	
	并发 共享 虚拟 异步
	
	并发 和 共享
		是多用户 多任务 的两个最基本的特征 也是互为存在的条件
	
```

```
	
	并行 与 并发 //相似但是有区别
	
		并行是两个或以上 多个事件在同一时间发生
		并发是两个或以上 多个事件在同一时间间隔内发生
	
	
	共享
		
		资源共享 资源复用 
        //系统的资源远远小于 多道程序的总和 所以会形成它们对共享资源的争夺
		
		实现资源共享的方式有两种
			互斥共享方式
				//打印机 虽然可以提供多个进程 但是在一段时间内 只允许一个进程访问该资源 所以应建立一种机制 保证多个进程对这类资源的互斥访问
				//A进程请求使用 且资源空闲 A使用 如果其他进程 再访问 需要等待A进程结束 才能进行访问 这称为互斥式共享 这种只允许一个进行访问的资源 称为临界资源(独占资源) //系统大多数物理设备 栈 变量 和 表格都属于 临界资源 计算机需要配置这种机制 保证进程互斥的使用 临界资源
				
			同时访问方式
				//宏观意义上 在一段时间允许 多个进程 同时访问 但是微观上 他们是交替运行	//内存 磁盘等
			
```

```
	
	虚拟
	
		通过某种技术 把物理实体 变为若干个逻辑上的对应
		用于实现虚拟的技术 称为虚拟技术
		OS 利用了两种方式 实现虚拟技术 
			时分复用技术
				分时使用方式 //利用设备的空闲时间 转去为其它用户服务 使设备得到充分的利用
				广泛利用该技术 实现 虚拟处理机 虚拟设备 提高资源利用率
				
			空分复用技术
				虚拟磁盘技术
				虚拟存储器技术
				//提高存储空间的利用率
				//将范围比较宽的信道 划分成多个频率范围较窄的信道
	
	
	
	异步 //也叫不确定性
		
		因为 系统允许多个进程并发执行 但只有一个处理机 每次只允许一个进程执行 其他进程只能等待

#### 处理机管理

```
	
		多道程序中处理机的分配和运行都是 以进程为基本单位的 所以对处理机的管理可以归结 为对进程的管理 
		主要功能     
            创建和撤销进程 进程控制
                创建和回收进程资源 一个进程创建若干线程 提高系统并发性
                
            对进程的运行进行协调 进程同步
            	//为了多个进程(线程)的运行进行协调 常见两种协调方式
            		进程互斥方式
            			给临界资源配置一把锁
            		进程同步方式
            			信号量机制实现
            			
			实现进程之间的信息交换 进程通信
				通常直接采用 直接通讯方式 //由源进程利用发送命令直接将消息 挂到目标进程的消息队列上 目标进程利用接受命令从消息队列中取出消息
				
			以及按照一定的算法把处理机分配给进程 分为两步
				包括作业调度(按照算法选出作业 分配资源 调入内存后 分别为它们建立进程 使他们都成为可能获得处理机的就绪进程 插入就绪队列中) 
				和 
				进程调度 从进程的就绪队列按照算法 选出一个进程 讲处理机分配给他 让其投入执行
	
```

#### 存储器管理

```
	
	内存分配
		为每到程序分配内存空间
		提高存储器利用率
		实现内存分配时 可以采取静态和动态两种方式
			静态分配
				内存空间在作业 装入时确定的 整个运行期间不允许在申请新的内存空间也不允许作业在内存中移动
			动态分配
				内存空间也是在作业 装入时确定 但是可以在运行时 申请增加附加内存空间 适应程序和数据的动态增长
	
	
	内存保护
		确保每道 用户程序 都仅在自己的内存空间内运行 互不干扰
			//设置内存保护机制 设置两个界限寄存器
			//系统对每条指令访问的地址进行检查 如果发生越界 就发出越界中断请求 越界检查都是由硬件实现的
		不允许用户程序访问 操作系统的程序和数据 也不允许用户程序的数据转移到非共享的其他的用户程序中
	
	
	地址映射
		将地址空间中的逻辑地址 转换为 内存空间的物理地址 硬件支持下完成
		
	
	内存扩充
		将借助于虚拟存储技术 从逻辑上扩充内存容量 使用户所感觉的内存容量比实际内存大得多	
		//需要实现 必须设置内存扩充机制(包含少量的硬件) 用于实现请求调入功能 置换功能

#### 进程的基本状态及转换

```
	
	多个进程 在并发执行时 共享系统资源 他们在运行过程中 呈现间断性规律 进程在生命周期内 可能具有多种状态
	
	就绪状态
		进程处于准备好 运行的状态 进程已分配到除进程以外的 所有必要资源后 只要再获得CPU便可 立即执行 
		//多个 按照策略 排成队列 就绪队列
		
	执行状态
		进程获得cpu 程序正在进行 单处理机只有一个程序运行 多处理机可以多个
		
	阻塞状态
		正在执行的进程 由于发生某事件 I/O请求 申请缓冲区失败 暂时无法继续执行时的状态 系统会把处理机分配给 其它的就绪进程 也称等待状态 封锁状态4
		
		
		
	创建状态 //只有一次
		//为了 保证对进程控制操作的完整性 保证调度必须在创建状态完成
	终止状态 //只有一次
		自然结束
		无法克服错误
		被OS终结
		被其它有终止权的进程所终结
		//进入终止状态 的进程以后不能再 执行
		
```

```
	
	运行状态(被占用临界资源) > 阻塞状态(I/O请求)
	
	阻塞状态 > 就绪状态(I/O完成)
	
	CPU调度 就绪状态 > 执行状态 
	
	并发执行(时间片用完 优先级更高的抢占CPU) 执行状态 > 就绪状态
	

	就绪 到 执行 不会引起其他 状态转换 
	运行 到 就绪 必然会引起其他进程 状态 要选出下一个就绪进程运行
	执行 到 阻塞 就绪状态非空 引发就绪状态执行 引发
	阻塞 到 执行 CPU空闲 没有其它进程竞争CPU 进程被立即调度 引发
		//也可能同时 引发状态 1和2 抢占CPU
		
		
	也有 挂起操作 进程处于静止状态 他将暂停执行 如果处于就绪状态 那么暂时不接受调度 等待激活操作
		
		

```
